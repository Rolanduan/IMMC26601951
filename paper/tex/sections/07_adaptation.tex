\section{Transfer by Park Morphology}
The new framework is intentionally transferable, but it should not be copied by park name. It should be transferred by morphology. What matters is whether a reserve behaves like a gateway-constrained system, a dispersed-entry system, or a seasonally fractured system.

\subsection{Three Transfer Classes}
We distinguish three broad classes:
\begin{itemize}
  \item \textbf{Gateway-constrained parks}: a small set of entries dominates access, so hinge protection and rapid interception matter most.
  \item \textbf{Dispersed-entry parks}: pressure enters diffusely, so the model must weight broader surveillance coverage more heavily than narrow bottleneck control.
  \item \textbf{Seasonally fractured parks}: mobility and sink structure shift by season, so the transport graph and service rates must be rebuilt on a cycle.
\end{itemize}

\subsection{Transfer Protocol}
The correct transfer routine is therefore:
\begin{enumerate}
  \item identify ingress nodes and classify park morphology;
  \item rebuild habitat sinks and the weighted value field;
  \item reconstruct the road or trail graph and re-estimate service rates by station;
  \item regenerate defense templates and recompute the stability frontier.
\end{enumerate}
This is a stronger rule than simply porting weights. It preserves the architecture while localizing the objects that actually drive the decision.

\subsection{Illustrative Reuse}
Yellowstone behaves more like a seasonally fractured system because winter closure, visitor movement, and seasonal wildlife concentration all change the transport graph materially \citep{yellowstoneOfficial}. Kakadu behaves more like a morphology-shifting park in which wet--dry transitions alter both ingress feasibility and service time \citep{kakaduOfficial}. In each case, the barrier-design logic still applies, but the hinge set and the service bottleneck move.
