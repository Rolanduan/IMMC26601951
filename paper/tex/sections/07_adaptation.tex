\section{Transfer by Liability Morphology}
The new framework is intentionally transferable, but it should not be copied park by park using one fixed map. It should be transferred by \emph{liability morphology}: the geometric relationship among biological value, incident uncertainty, and service capacity.

\subsection{Three Transfer Classes}
We distinguish three broad classes:
\begin{itemize}
  \item \textbf{Boundary-loaded parks}: liability is driven mainly by a few exposed borders or gates, so contract cells cluster near ingress edges.
  \item \textbf{Sink-clustered parks}: ecological value is concentrated around a few habitat clusters, so contract cells cluster around those sinks rather than around one entry axis.
  \item \textbf{Seasonally bifurcated parks}: wildlife value, access, and workload structure all shift by season, so the liability surface must be rebuilt cyclically.
\end{itemize}

\subsection{Transfer Protocol}
The correct transfer routine is therefore:
\begin{enumerate}
  \item rebuild the ecological-consequence layer \(V_i\) from local species and habitat data;
  \item rebuild the latent incident field and the information-deficit field;
  \item rebuild station workload means and variances for the new service system;
  \item regenerate the contract set \(\mathcal{C}\) and solve the portfolio model again.
\end{enumerate}
This is a stronger rule than porting weights. It preserves the logic of the contract while localizing the objects that actually determine where failure is expensive.

\subsection{Illustrative Reuse}
Yellowstone behaves more like a seasonally bifurcated system because winter closure, visitor circulation, and wildlife concentration all change the liability field materially \citep{yellowstoneOfficial}. Kakadu behaves more like a morphology-shifting system because wet--dry transitions alter both access and service capacity \citep{kakaduOfficial}. In each case, the contract logic still transfers, but the contract cells and workload guarantees must be rebuilt from scratch.
