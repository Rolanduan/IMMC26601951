\section{Strengths, Identification Risks, and Adversarial Failure}
\subsection{Why the New Formulation Is Stronger}
The re-derived framework has four structural advantages over a plain hotspot-allocation approach.
\begin{itemize}
  \item It changes the decision object from generic spatial risk to weighted loss pathways, which is closer to how protection actually fails.
  \item It distinguishes between controlling average exposure and controlling the worst hinge, which prevents visually attractive but operationally fragile plans.
  \item It ties staffing to service reliability through station load rather than only through aggregate patrol hours.
  \item It offers a cleaner transfer rule across parks because ingress structure, habitat sinks, and service rates are easier to rebuild than an entire bespoke patrol doctrine.
\end{itemize}

\subsection{Identification Boundary}
The framework also has nontrivial identification risk.
\begin{itemize}
  \item If ingress priors are badly misspecified, the model can overprotect one gateway while underprotecting an unseen alternative route.
  \item If the road graph misses informal tracks or off-road movement, hinge criticality can be systematically biased.
  \item If service times are calibrated from optimistic patrol assumptions, the queueing layer understates delay exceedance and therefore understates staffing needs.
  \item If wildlife value is seasonally volatile, a static sink map can misidentify which pathways are actually expensive to defend this month.
\end{itemize}

The sharpest failure mode is adversarial rerouting. Once defenders repeatedly seal the same obvious hinge, pressure may jump to a second-best route that was previously subcritical. That is exactly why the model should be run as a rolling decision system rather than as a one-time map product.

\subsection{Appropriate Validation Standard}
This paper should therefore be read as a decision-support architecture, not as a definitive forecast. The correct validation routine is to compare predicted hinge use with observed signals, predicted delay exceedance with realized dispatch logs, and predicted reserve value with actual incident absorption. If those diagnostics drift, the transport graph or the service model must be rebuilt before the deployment logic is trusted again.
